# supply_integration — Domain Glossary

## Supplier
A promotional products vendor (e.g. MKO/Makito, XDC). Each supplier has its own API,
data structure, and supplier_reference format. The codebase is designed to support many
suppliers in parallel.

## SupplierConfig
A dataclass that holds everything needed to extract one supplier's data: name, base_url,
and a dict of endpoint suffixes keyed by feed type (product, price, stock, print, print_price).

## SupplierExtractor
The abstract seam for extraction. Each supplier has one concrete adapter (e.g. MkoExtractor)
that implements `run(date)`. The orchestrator iterates over a list of extractors.

## Feed
One data stream from a supplier API (e.g. product catalog, pricing, stock, print options).
Each feed produces one or more Parquet files on S3 under `{supplier}/raw/{feed}/{date}/`.

## Staging model
A dbt view (prefix `stg_`) that reads raw Parquet from S3 and applies type casting,
renaming, and light cleaning for one supplier. No business logic.

## Intermediate model
A dbt view (prefix `int_`) that normalises a staging model to the canonical column shape
and adds the `supplier` column. One intermediate model per supplier per feed.

## Canonical model
A dbt mart table (no prefix) that unions all intermediate models for a given feed type
(e.g. `catalog`, `variants`, `prices`, `print_options`). The UI and downstream systems
query canonical models, not supplier-specific ones.

## Supplier reference
A structured string sent to the supplier when placing an order. Format is supplier-specific
and may also depend on product_type. Generated by `ui/supplier_reference.py`.
- MKO corporate gifts (no print): `variant_id`
- MKO with print: `variant_id__teccode#areacode[__teccode#areacode ...]`
- XDC clothing: `size[gender_code]=variant_id` (future)

## PCM template
A named category template (e.g. "Bag - DrawstringV3") used in Category Management.
Products are configured within a template session. Each template maps to a set of
products and defines quantity step rules.

## Quantity code
A string encoding the quantity steps displayed on the website for a product, e.g.
`1-1:1,5-50:5,50-100:10,100-250:25,250-500:50`. Format: `min-max:step` pairs.

## Slug
A URL-safe product identifier derived from the product name:
`lower(REGEXREPLACE(name, "[^0-9a-zA-Z]", ""))`. Generated automatically.

## Bespoke
The internal Helloprint team workflow for configuring one-off supplier orders:
select product → variant → print options → carrier → generate supplier_reference.

## Category Management (catman)
The internal team workflow for configuring which products appear on the website:
select products by supplier/template, assign slugs and quantity codes,
select print options (always max dimensions), export a PCM CSV.
